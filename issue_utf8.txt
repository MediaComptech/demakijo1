# Rencana Migrasi ke Native PHP MVC (Sistem Informasi Akademik SDN Demakijo 1)

Dokumen ini adalah cetak biru (blueprint) untuk melakukan konversi penuh dari framework Laravel ke arsitektur **Native PHP MVC** yang ringan, modern, dan mudah dipelihara (maintainable) oleh *junior programmer* maupun model AI berbiaya rendah.

## Latar Belakang & Tujuan
Tujuan utama migrasi ini adalah menyederhanakan kompleksitas aplikasi dengan membuang *magic code* dan abstraksi yang terlalu dalam, tanpa mengorbankan fitur, keamanan, maupun performa. Aplikasi akan menggunakan komponen mandiri (seperti Standalone Eloquent dan BladeOne) yang dirangkai dalam arsitektur MVC sederhana buatan sendiri (Custom Core Framework).

## Rencana Fase Implementasi

Proses konversi akan dibagi menjadi beberapa fase agar terstruktur dan mudah di-*debug*:

### Fase 1: Setup Struktur Folder & Dependency (Composer)
- Membuat struktur direktori baru: `app/`, `config/`, `public/`, `routes/`, `storage/`, `vendor/`.
- - Inisialisasi `composer.json` dengan dependency minimal:
- -   - `illuminate/database` (ORM)
- -   -   - `eftec/bladeone` (Template Engine)
- -   -   -   - `vlucas/phpdotenv` (Env parser)
### Fase 2: Pembuatan Core Framework (`app/Core/`)
Membangun pondasi Native MVC yang *explicit* dan mudah dibaca:
- `App.php`: Bootstrap aplikasi dan dependency injection sederhana.
- - `Router.php`: Sistem routing native berbasis *closure* dan string `Controller@method`.
- - - `Controller.php`: Base controller dengan fungsi helper `view()` dan `json()`.
- - - - `Model.php`: Wrapper/inisialisasi koneksi Eloquent Standalone.
- - - - - `View.php`: Wrapper untuk integrasi dengan BladeOne.
- - - - - - `Security.php`: Proteksi CSRF, XSS filter, dan sanitasi input.
- - - - - - - `Session.php` & `Auth.php`: Manajemen sesi aman dan login/role check.
- - - - - - - - `Validator.php`: Validasi input sederhana tanpa abstraksi rumit.
### Fase 3: Migrasi Konfigurasi & Helper
- Menyalin `.env` dan membuat file `config/database.php`, `config/app.php`.
- - Membuat file helper global (`app/Helpers/GlobalHelper.php`) untuk fungsi seperti `url()`, `asset()`, `csrf_token()`, `redirect()`.
### Fase 4: Migrasi Model & Database
- Menyalin seluruh file Model dari Laravel ke `app/Models/`.
- - Memastikan namespace disesuaikan (misal `App\Models`).
- - - Karena kita menggunakan Standalone Eloquent, struktur relasi tidak perlu diubah.

### Fase 5: Migrasi Routing
- Mengonversi `routes/web.php` dan `routes/api.php` dari sintaks Laravel Facade ke native router instance.
-   *Contoh:* `$router->get('/dashboard', 'DashboardController@index');`

### Fase 6: Dokumentasi Lengkap
- Membuat panduan komprehensif bagi *junior programmer* di dalam `README.md` dan folder `docs/`.
